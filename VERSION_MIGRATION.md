# 版本号迁移说明

## 旧体系 → 新体系

| 旧标识 | 新标识 | 说明 |
|--------|--------|------|
| GitHub Release **v1.3.0** | **v1.0.3** 起 | 主版本固定 **1.0**，补丁用 **1.0.x** |
| PyPI / `pyproject.toml` **1.3.0** | **1.0.3** | 与 Release 对齐 |
| AstrBot 插件 **v2.2.0** / **2.0.0** | **1.0.3** | 与主包同一 **1.0.x** 线 |
| `install.bat` + PyInstaller **~7MB** exe | **sts2skill.exe** 内嵌 payload 解压安装 | 见 README |

**1.0.3** 在功能上承接原 **1.3.0** 本地未发布内容（角色编号 0–4、AstrBot WebUI schema、解压式安装程序），仅**重新编号**，不表示功能回退。

## 升级时你需要做什么

1. **已装 v1.3.0 / 旧插件**：在 AstrBot 插件管理中重载 `astrbot_plugin_sts2_agent`，或重新运行 **`sts2skill.exe`** 选择 AstrBot 并指向你的 `data` 目录。
2. **配置**：`character` 推荐使用 **0–4** 整数；旧字符串 `ironclad` 等仍可由桥接解析，但 WebUI 已改为整数表单项。
3. **Release 标签**：GitHub 上 **v1.3.0** 标签保留作历史；新发布请用 **v1.0.4**、**v1.0.5** … 递增。

## 版本递增约定

- **1.0.x**：补丁（修复、文档、安装器、AstrBot schema）
- 若将来有破坏性变更，再考虑 **1.1.0**（仍保持 1.x 主版本，不做 2.x / 3.x 跳跃）
